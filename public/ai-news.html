如何接入真实数据

目前使用本地 mock 数据，以下是接入真实 AI 抓取的步骤

01

替换数据源

将 ai-news.js 里的 MOCK_NEWS 替换成真实 API 调用。 推荐使用 Tavily Search API（专为 AI agent 设计，支持按关键词抓取最新结果） 或 Gemini Grounding API（直接从搜索结果中提炼摘要）。 

// 示例：调用 Tavily
const res = await fetch('https://api.tavily.com/search', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    api_key: 'YOUR_TAVILY_KEY',
    query: 'AI news today',
    search_depth: 'advanced',
    max_results: 10
  })
});
const data = await res.json();
// data.results → 解析成 NewsItem[]
02

用 AI 生成摘要与分级

把抓取的原始文章标题 + 摘要丢给大模型，让它按"必读/推荐/了解"打分并输出 3 句中文摘要。 

// 示例 Prompt（可接入 Gemini / OpenAI）
const prompt = `
你是一名 AI 行业分析师。
对以下新闻按重要程度分级（must/rec/info）并
用 3 句中文写出核心摘要，返回 JSON。

新闻：${article.title}
内容：${article.content}
`;
// 解析 JSON → 填入卡片数据
03

定时自动更新（Vercel Cron）

如果未来迁移到 Next.js / 部署到 Vercel，可在 vercel.json 里配置 Cron Job， 每天早 8 点自动抓取并写入 KV 存储或静态 JSON 文件。 

// vercel.json（迁移到 Next.js 后使用）
{
  "crons": [{
    "path": "/api/fetch-ai-news",
    "schedule": "0 0 * * *"  // UTC 0点 = 北京时间 8点
  }]
}

04

现在怎么手动更新

在 ai-news.js 顶部的 MOCK_NEWS 数组里增减条目， 每条对象的字段：level（must/rec/info）、title、 source、points（摘要数组，3条）、url。 

{
  level: 'must',               // must | rec | info
  title: '文章标题',
  source: '来源名称',
  sourceUrl: 'https://...',    // 可选：来源主页
  points: [
    '第一条摘要',
    '第二条摘要',
    '第三条摘要',
  ],
  url: 'https://原文链接',
  date: '2026-03-10',          // 可选：发布日期
}
